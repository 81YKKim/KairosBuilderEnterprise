# Memory Layer

## Role

The Memory layer stores accepted knowledge, decisions, lessons, and evolution history.

Memory is curated. It is not a dump of every observation.

## Inputs

- Accepted knowledge entities
- Decision records
- Validation records
- Lessons approved for reuse
- Architecture updates
- Sprint outcomes

## Responsibilities

- Preserve durable context.
- Keep evidence references.
- Support future reasoning and planning.
- Avoid stale or unvalidated assumptions.
- Separate temporary observations from accepted memory.

## Outputs

- Memory records
- Historical references
- Accepted lessons
- Architecture history

## Constraints

- Memory must be traceable.
- Memory must not override repository source of truth.
- Memory must be updateable when repository evidence changes.
