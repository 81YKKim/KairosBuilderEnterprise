# Learning Layer

## Role

The Learning layer extracts stable lessons from validated outcomes.

Learning is not uncontrolled self-modification. It proposes durable patterns only after evidence supports them.

## Inputs

- Validation outcomes
- Decision records
- Execution results
- QA issue trends
- Impact outcomes
- Human feedback

## Responsibilities

- Identify repeated patterns.
- Distinguish one-off fixes from reusable lessons.
- Propose improvements to Knowledge and Memory.
- Avoid learning from failed or unvalidated work unless marked as failure evidence.

## Outputs

- Lessons learned
- Pattern candidates
- Process improvement candidates
- Architecture feedback

## Constraints

- Must not silently change policy or execution behavior.
- Must remain evidence-linked.
- Must support human review before durable adoption.
